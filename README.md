# AIニュース収集アプリ

React + TypeScript + Vite のフロントエンドと、Google News RSS を中継する Vercel Functions を使った、自分用の AI ニュース収集アプリです。  
白ベースのシンプルなUIで、キーワード収集、除外条件、期間フィルタを使いながら記事一覧を見やすく確認できる最小構成になっています。

## 概要

- ニュースソースは Google News RSS
- ブラウザから直接 RSS を叩かず、`/api/news` の Vercel Function 経由で取得
- 本文表示は `/api/article` の Vercel Function 経由で取得
- キーワードごとに記事を収集し、簡易な重複統合を実施
- 除外ワード、除外ソース、期間フィルタで一覧を絞り込み
- 気になった記事は「後で読む」に保存可能

## 主な機能

- 複数キーワードの登録
- キーワードごとの ON / OFF 切り替え
- AI ニュース記事の一覧表示
- 表示項目:
  - タイトル
  - 媒体名
  - 公開日時
  - 要約
  - 元記事リンク
- 記事カード内での本文折りたたみ表示
  - 開いたときだけ本文を取得する遅延読み込み
  - Google News の誘導URLを元記事URLへデコードしてから本文抽出
- 重複っぽい記事の簡易統合
  - タイトル完全一致
  - 記号除去 + 空白正規化 + 小文字化後の一致
- 除外ワード機能
  - タイトルまたは要約に含まれる記事を非表示
- ソース名ごとの除外機能
- 期間フィルタ
  - `24時間以内`
  - `3日以内`
  - `7日以内`
- 「後で読む」保存
  - 現在のUIではパネル非表示
  - 保存機能自体は残してあり、再表示可能な構成
- 条件設定パネル
  - 登録済みキーワード一覧
  - 除外ワード一覧
  - 除外ソース一覧

## ローカル起動手順

Node.js / npm が使える前提:

```bash
npm install
npm run dev
```

Vite 単体の起動先:

```text
http://localhost:5173
```

この起動方法はフロントのみの確認向けです。API 経由のニュース取得確認は Vercel 側の起動を使ってください。

## Vercel での起動方法

API を含めて動かす場合:

```bash
npx vercel dev
```

または:

```bash
npm run dev:vercel
```

起動先:

```text
http://localhost:3000
```

補足:

- 初回は Vercel CLI のログインが必要になることがあります
- このリポジトリでは `/api/news` と `/api/article` を Vercel Functions として使います

## localStorage に保存している項目

- `ai-news-keywords`
  - 登録済みキーワード一覧
- `ai-news-keyword-enabled-map`
  - キーワードごとの ON / OFF 状態
- `ai-news-excluded-words`
  - 除外ワード一覧
- `ai-news-excluded-sources`
  - 除外ソース一覧
- `ai-news-period-filter`
  - 選択中の期間フィルタ
- `ai-news-saved-articles`
  - 「後で読む」に保存した記事

## 現在のUI方針

- 白ベース、薄いグレーの境界、柔らかい影で構成
- ニュース一覧を主役にし、情報量は多くても圧迫感が出すぎないように調整
- 通常時は入力と再取得だけ見せ、詳細な管理項目は折りたたみの「条件設定」に集約
- 記事一覧ヘッダでは、表示件数・除外件数・期間をコンパクトに把握できるように配置
- 記事本文はカード内の折りたたみパネルとして表示
- 「後で読む」は現時点では通常非表示にして、一覧の読みやすさを優先

## 既知の制約

- Google News RSS の結果品質や description は媒体依存
- 要約は RSS `description` の整形結果であり、記事本文の要約ではない
- 記事本文抽出は媒体ごとのHTML構造に依存するため、失敗する記事がある
- 元記事URLは Google News の誘導URLをデコードして取得しているが、すべての媒体で完全ではない可能性がある
- 重複統合は簡易版のみで、意味的に近い別タイトルまでは統合していない
- 期間フィルタは現状クライアント側の表示絞り込み
- ソース名表記ゆれには完全対応していない
- 「後で読む」は保存されるが、現在の通常UIでは表示パネルを隠している
